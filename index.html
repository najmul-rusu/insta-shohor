Question and Answer

Web browsers load a web page, parse HTML, and create content known as the Document Object Model (DOM). DOM presents a live view of web pages in your JavaScript code. The browser will then grab everything linked to the HTML, such as images and CSS files. CSS information comes from the CSS parser. HTML and CSS are first combined by DOM to create web pages. Then, the browser's JavaScript engine loads the JavaScript file and inline code but does not execute the code immediately. It waits for HTML and CSS loading to finish. Once this is done, JavaScript is executed as the code is written. This allows the DOM to be updated by JavaScript code and rendered by the browser.

The common denominator in all environments is a built-in mechanism called event loop, which manages the execution of multiple parts of your program over time, each time calling the JS engine. This means that JS Engine is an on-demand execution environment for any arbitrary JS code. It is the surrounding environment that scheduling events (JS code execution). Now some requests the host system is processing will depend on the host, but the JavaScript engine will never process more than one statement at a time.
